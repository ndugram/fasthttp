# Безопасность

FastHTTP включает встроенную систему безопасности, которая работает под капотом и не требует дополнительной настройки. Все проверки выполняются автоматически для каждого запроса и ответа.

## Как это работает

Система безопасности интегрирована в ядро библиотеки. При каждом запросе:

```
Запрос пользователя
        ↓
Проверка URL (SSRF защита)
        ↓
Проверка заголовков
        ↓
Выполнение запроса
        ↓
Проверка ответа
        ↓
Возврат результата
```

Вам не нужно ничего настраивать или добавлять — защита работает автоматически.

## Что защищено

### Защита от SSRF атак

SSRF (Server-Side Request Forgery) — это атака, при которой злоумышленник заставляет сервер делать запросы к внутренним ресурсам.

FastHTTP автоматически блокирует запросы к:

- `localhost` и его варианты
- `127.0.0.1`, `0.0.0.0`, `::1`
- Приватные IP-адреса: `10.x.x.x`, `192.168.x.x`, `172.16-31.x.x`
- Link-local адреса: `169.254.x.x`
- Домены: `.local`, `.intranet`, `.internal`

```python
from fasthttp import FastHTTP

app = FastHTTP()

# Этот запрос будет заблокирован автоматически
@app.get(url="http://localhost:8080/admin")
async def blocked_request(resp):
    return resp.json()

app.run()
# Результат: SSRF blocked
```

### Маскирование секретов в логах

При включённом режиме отладки FastHTTP автоматически скрывает конфиденциальные данные в логах:

- Заголовки `Authorization`, `Cookie`, `X-API-Key`
- Параметры URL с `api_key`, `token`, `password`

```
# Вместо:
Authorization: Bearer sk-1234567890abcdef

# В логах вы увидите:
Authorization: *****
```

### Circuit Breaker

Если хост перестаёт отвечать (много таймаутов или ошибок), FastHTTP автоматически перестаёт слать запросы к этому хосту на некоторое время. Это защищает от:

- Заваливания упавшего сервиса запросами
- Траты ресурсов на неработающие хосты
- Каскадных отказов

После периода ожидания FastHTTP периодически проверяет, восстановился ли хост.

### Ограничение размера ответа

По умолчанию FastHTTP ограничивает размер ответа до 100 МБ. Это защищает от:

- Атак с гигантскими ответами
- Утечек памяти
- Зависания приложения

### Защита заголовков

FastHTTP автоматически:

- Санитизирует заголовки от CRLF-символов (защита от HTTP Response Splitting)
- Проверяет входящие заголовки на подозрительные значения
- Проверяет `Set-Cookie` на безопасные флаги

### Таймауты

Все запросы имеют встроенные таймауты:

- Таймаут соединения: 10 секунд
- Таймаут запроса: 30 секунд

Это предотвращает зависание приложения при проблемах с сетью.

### Защита от редиректов

FastHTTP ограничивает количество редиректов (по умолчанию 10) и блокирует опасные перенаправления:

- На `file://` протокол
- На внутренние IP-адреса
- На `javascript:` или `data:` URL
- HTTP Downgrade (HTTPS → HTTP)

### Ограничение параллельных запросов

По умолчанию FastHTTP ограничивает количество одновременных запросов до 100. Это защищает от случайного создания слишком большого количества соединений.

## Отключение защиты

По умолчанию все проверки включены. Это рекомендуемая конфигурация для большинства случаев. При необходимости защиту можно отключить:

```python
from fasthttp import FastHTTP

app = FastHTTP(security=False)
```

Отключение защиты не рекомендуется, если у вас нет веских причин.

## Логирование событий безопасности

При блокировке запроса вы увидите сообщение в логах:

```
ERROR | SSRF blocked: SSRF protection blocked request to: http://localhost/test
ERROR | Circuit breaker open for: api.example.com
ERROR | Security error: Response too large: 150MB
```

Включите режим отладки для более подробной информации:

```python
app = FastHTTP(debug=True)
app.run()
```

## Лучшие практики

1. **Не отключайте защиту** — встроенные механизмы защищают от распространённых атак
2. **Используйте HTTPS** — для защиты данных при передаче
3. **Обрабатывайте ошибки** — проверяйте返回值 на None при ошибках
4. **Следите за логами** — регулярно проверяйте логи на предмет заблокированных запросов

## Безопасность по умолчанию

Главное преимущество системы безопасности FastHTTP — она работает «из коробки». Вам не нужно ничего настраивать, чтобы получить защиту от:

- SSRF атак
- Утечки секретов в логах
- Перегрузки от неработающих хостов
- Слишком больших ответов
- Опасных редиректов
- Зависания запросов

Просто используйте библиотеку — безопасность уже встроена.
