# Circuit Breaker

Автоматическая защита от падающих хостов.

## Как это работает

Если хост перестает отвечать (множественные таймауты или ошибки), FastHTTP временно перестает отправлять запросы к этому хосту.

## Защита от

- Заваливания упавшего сервиса запросами
- Траты ресурсов на неработающие хосты
- Каскадных отказов

## Поведение

1. Хост испытывает множественные ошибки
2. Circuit breaker открывается
3. Запросы к хосту блокируются
4. После периода ожидания FastHTTP проверяет восстановление
5. Если восстановлен, нормальная работа возобновляется

## Логирование

```
ERROR | Circuit breaker open for: api.example.com
```
